# AgentResult Growth Control Customer Demo Contract

Этот документ фиксирует повторяемое customer demo. Его цель - показать собственнику не “AI-кабинет”, а рабочий контур:

```text
AgentResult готовит -> собственник принимает решение -> материал передаётся в выпуск -> выход подтверждается -> результат фиксируется
```

## Reset Contract

`?demo=pilot` сбрасывает только browser-state демо: язык, локальные решения, локальные передачи и временные действия в текущем браузере. Он не меняет seed-данные backend.

Backend seed сбрасывается только отдельной командой для demo tenant:

```bash
AI_GROWTH_OS_STORAGE=postgres \
DATABASE_URL=postgres://... \
npm run demo:reset-pilot
```

VPS-формат:

```bash
docker run --rm \
  --network agentresult-os-net \
  --env-file /opt/agentresult-os/app/.env \
  agentresult-os-backend:<sha> \
  node apps/backend/dist/db/reset-pilot-demo.js
```

Seed после reset должен давать:

- `Ждёт решения`: 1
- `Передано вручную`: 0
- `Вышло`: 1
- `Заявки`: 3
- один материал на согласовании: `Пост: контроль выпуска без ручного хаоса`

Контроль API:

```bash
curl -s -H 'x-tenant-id: 10000000-0000-4000-8000-000000000001' \
  https://dashboard-orpin-mu-26.vercel.app/api/agentresult-os-demo/approvals
```

Ожидание: один `pending` approval.

## 5-Minute Web Demo

### 0. Opening

Сказать:

```text
Покажу не кабинет с метриками, а контур контроля. Здесь видно, что готово, что требует решения, что передано в выпуск, что вышло и какой сигнал появился.
```

Не говорить:

```text
AI сам управляет ростом.
```

### 1. Сегодня

Open:

```text
https://dashboard-orpin-mu-26.vercel.app/?demo=pilot#/overview
```

Показать:

- `Главное`
- цепочку `Подготовлено -> согласовано -> выпущено -> сигнал`
- активный шаг `Решение`
- `Вышло: 1`
- `Сигнал: 3 заявки`

Сказать:

```text
Это ежедневный экран собственника. Он не объясняет систему, а показывает, где требуется решение и где уже есть факт результата.
```

Клик:

```text
Открыть решение
```

Ожидаемое состояние:

- открыт раздел `Публикации`
- вкладка `На согласование`
- видно `1 ждут решения`
- доступны действия `Согласовать`, `Нужны правки`, `Отклонить`

### 2. Решение

Сказать:

```text
Публичный материал не уходит дальше без решения собственника. Можно согласовать, вернуть на правки или отклонить.
```

Клик:

```text
Согласовать
```

Ожидаемое состояние:

- открывается модальное окно решения
- комментарий необязателен
- после подтверждения очередь становится `0 ждут решения`
- в `Сегодня` шаг `Решение` становится чистым
- шаг `Выпуск` показывает `Готово к ручной передаче`
- появляется действие `Отметить передачу`

Важно:

```text
Согласование не публикует материал автоматически. Оно только открывает следующий безопасный шаг: передать в выпуск.
```

### 3. Альтернативные решения

Проверять в отдельном чистом demo-run через `?demo=pilot`.

Клик:

```text
Нужны правки
```

Ожидаемое состояние:

- требуется комментарий
- после отправки очередь становится `0 ждут решения`
- backend-admin текст, terminal-логи и технические ошибки не появляются

Клик:

```text
Отклонить
```

Ожидаемое состояние:

- требуется комментарий
- после отправки очередь становится `0 ждут решения`
- материал не считается выпущенным

### 4. Публикации

Open:

```text
#/publications
```

Показать:

- `На согласование`
- `План публикаций`
- `Пакет материалов`

Сказать:

```text
Здесь разделены три состояния: ждёт решения, передано вручную и вышло. Передано не равно опубликовано.
```

Клик:

```text
План публикаций
```

Ожидаемое состояние:

- виден выпускной план
- опубликованный материал остаётся `вышло`
- согласованный материал появляется как готовый к ручной передаче, если был согласован в текущем browser-run

### 5. Результаты

Open:

```text
#/analytics
```

Показать:

- `Вышло`
- `Сигнал`
- `Следующий шаг`

Клик:

```text
Открыть публикации
```

Ожидаемое состояние:

- открывается `Публикации -> План публикаций`

Вернуться:

```text
#/analytics
```

Клик:

```text
Проверить источник
```

Ожидаемое состояние:

- появляется короткий read-only статус: `3 заявки зафиксированы...`
- не открывается форма редактирования метрик
- демо не создаёт новое состояние

Сказать:

```text
Результат здесь - не обещание роста. Это факт: что вышло, какой сигнал пришёл и какой следующий шаг нужен.
```

### 6. Close

Сказать:

```text
Первый пилот не про полную автономность. Он про устойчивый контур: готово, решение, передача, подтверждение выхода и результат. После этого можно подключать Telegram, CRM, формы и реальные источники заявок.
```

## Click Checklist

Перед показом проверить:

- `Открыть решение` ведёт в `Публикации` и показывает pending approval.
- `Согласовать` закрывает решение и открывает ручную передачу, не публикацию.
- `Нужны правки` требует комментарий и закрывает решение без технического текста.
- `Отклонить` требует комментарий и закрывает решение без технического текста.
- `Открыть публикации` из `Результаты` ведёт в план публикаций.
- `Проверить источник` показывает read-only статус и не открывает форму.
- Mobile 390px: нет горизонтального overflow, decision modal помещается.
