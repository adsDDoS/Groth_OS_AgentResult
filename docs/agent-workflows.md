# Agent Workflows

Agents are task workers. They receive context, produce artifacts, and hand results back to the backend.

## Demand Map Generation

1. Load Offer Brain.
2. Identify product, pain, industry, comparison, integration, glossary, calculator, case, FAQ, alternative, and region opportunities.
3. Reject near-duplicate page ideas.
4. Add evidence requirements.
5. Create `demand_map_items`.

## SEO/GEO Brief

1. Select a demand map item.
2. Define audience, search intent, AI-search questions, proof needs, outline, internal links, and schema suggestions.
3. Save `content_briefs`, `schema_recommendations`, and `ai_answer_blocks`.
4. Flag unsupported claims.

## Draft Creation

1. Use approved or reviewed brief.
2. Pull Offer Brain, proof points, forbidden claims, tone rules, and related cases.
3. Draft useful B2B content.
4. Save `content_drafts`.
5. Move content item to `review`.

## Repurposing

1. Start from a reviewed or approved source idea.
2. Adapt to channel format.
3. Keep claims consistent.
4. Produce channel-specific drafts.
5. Do not publish.

## Proof Engine

1. Collect before/after story.
2. Extract metrics and source notes.
3. Mark public-use approval status.
4. Produce case narrative and reusable snippets.

## Analytics Feedback

1. Import page/channel metrics.
2. Detect stale, declining, or high-opportunity assets.
3. Create improvement tasks.
4. Route major live updates through approval.
