# GrothOS Pilot Call Packet

Access: `operator-only`

Использовать во время первого клиентского созвона. Держать показ в 7 минутах.

Квалифицировать до показа:

```text
docs/pilot-qualification-note.md
```

После `идём в пилот` собрать:

```text
docs/pilot-onboarding-intake.md
```

Dashboard:

```text
https://dashboard-orpin-mu-26.vercel.app/?demo=pilot&v=precall-dryrun#/overview
```

## Opening

```text
Я покажу GrothOS как рабочий контур текстовой контент-фермы: тема, решение, QA, выпуск, подтверждённая публикация и следующий контент-шаг.
```

## Transition 1: Сегодня

Открыть:

```text
https://dashboard-orpin-mu-26.vercel.app/?demo=pilot&v=precall-dryrun#/overview
```

Сказать:

```text
Это ежедневный пульт. Здесь видно не весь продукт сразу, а главное: что требует решения, где стоит выпуск и что уже вернулось результатом.
```

Показать:

- главное действие
- статусные ячейки
- очередь действий

Переход:

```text
Теперь покажу, как материал проходит от решения к выходу.
```

## Transition 2: Публикации

Открыть:

```text
https://dashboard-orpin-mu-26.vercel.app/?demo=pilot&v=precall-dryrun#/publications
```

Сказать:

```text
Здесь три состояния выпуска: очередь, проверка выхода и опубликовано. Важно: передано в выпуск не равно опубликовано.
```

Показать:

- решение собственника
- очередь выпуска
- проверка выхода
- счётчик опубликованного

Переход:

```text
После выхода смотрим на производственный результат публикации: что вышло, где вышло и что делаем дальше.
```

## Transition 3: Результаты

Открыть:

```text
https://dashboard-orpin-mu-26.vercel.app/?demo=pilot&v=precall-dryrun#/analytics
```

Сказать:

```text
Результат здесь — это URL, канал, формат, первичные реакции и решение, что делать с материалом дальше.
```

Показать:

- URL
- первичные реакции
- следующий контент-шаг

## Closing

```text
Первый пилот должен доказать управляемость производства: тема не теряется, текст не зависает, выпуск подтверждается, а результат превращается в следующий контент-шаг.
```

## Follow-Up Message

```text
Спасибо за разговор. Коротко зафиксирую, что показал:

1. GrothOS ведёт производство текста как контур: тема -> решение -> QA -> выпуск -> подтверждённая публикация -> следующий шаг.
2. В первом пилоте проверяем управляемость: материалы не теряются, выпуск подтверждается URL, первичная реакция фиксируется.
3. Рабочий сценарий пилота: выбрать 1-2 канала, завести недельный пакет тем, провести материалы через согласование/QA/выпуск и собрать первые результаты публикаций.

Следующий шаг: выбрать пилотный канал и один тип материалов, на котором проверим контур за первую неделю.
```

## Не Заходить

- `База`
- `Настройки`
- backend/API/VPS/GitHub/Vercel
- reset commands
- raw tokens or env values
- Telegram unless asked
- leads, CRM, revenue attribution, guaranteed growth
