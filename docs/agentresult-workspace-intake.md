# AgentResult Workspace Intake

Use this checklist to fill the local AgentResult workspace with real company data.

## 1. Company

Fill these fields:

- Company name
- Main website
- Short positioning
- Who you sell to
- Which problems you solve
- What proof you can already show
- What you must not promise
- Tone of voice
- Main competitors and alternatives

Copy-friendly format:

```text
Company name:
Website:
Positioning:
Who we sell to:
Problems we solve:
Proof we can use:
What we must not promise:
How we speak:
Competitors and alternatives:
```

## 2. Products

For each product or service:

- Name
- Category
- Short description
- Main result for the client
- Who it is for
- What makes it different
- Which integrations matter

Copy-friendly format:

```text
Product name:
Category:
Description:
Main result:
Who it is for:
Differentiators:
Important integrations:
```

## 4. Tools

For each tool already used or planned:

- Tool name
- Tool type
- Current status
- Whether the client already works in it
- URL or endpoint
- Access owner
- What the system may do
- What the system must not do

Allowed tool types:

- website
- social
- CRM
- analytics
- email
- spreadsheet
- other

Allowed statuses:

- connected
- needs setup
- not used
- later

Allowed permissions:

- read only
- prepare materials
- send for approval
- publish after approval

Copy-friendly format:

```text
Tool name:
Tool type:
Current status:
Client already uses it: yes/no
URL or endpoint:
Access owner:
Allowed actions:
Limits:
```
